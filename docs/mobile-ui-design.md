# 移动端 UI 设计文档

本文档只描述移动端页面布局、页面结构、展示内容和交互入口，不约束视觉风格、颜色、字体、图标样式。

## 整体页面结构

移动端建议使用固定底部导航，核心页面放在一级导航中。

底部导航：

```txt
矿机
市场
团队
资产
我的
```

通用页面结构：

```txt
[顶部栏：页面标题 / 钱包状态 / 必要操作]
[页面主体内容]
[底部导航]
```

通用列表结构：

```txt
[筛选区]
[列表卡片]
[加载更多 / 空状态]
```

通用弹窗结构：

```txt
[弹窗标题]
[核心信息]
[表单项]
[确认按钮]
[取消按钮]
```

## 1. 登录 / 注册页

### 布局

```txt
[产品名称 / Logo]
[连接钱包按钮]

连接后：
[钱包地址]
[登录 / 注册状态提示]
[邀请码输入框 - 仅注册时展示]
[登录 / 注册按钮]
```

### 展示内容

- 当前钱包连接状态。
- 当前钱包地址。
- 账户是否已注册。
- 邀请码输入框。
- 登录或注册按钮。
- 签名状态提示。

### 交互

1. 连接钱包。
2. 调用 `GET /auth/account/:address/exists` 判断账户是否存在。
3. 已存在时展示登录按钮。
4. 不存在时展示邀请码输入框和注册按钮。
5. 获取 nonce。
6. 钱包签名。
7. 登录或注册成功后进入主页面。

## 2. 矿机页

### 布局

```txt
[顶部概览]
账户余额
VIP等级 / 节点等级

[矿机列表]
[矿机卡片]

```

### 矿机卡片展示内容

- 矿机名称。
- 价格。
- 最大产出奖励。
- 是否可购买。
- 用户是否已拥有。
- 已产出奖励。
- 当前周期结束时间。
- 每秒奖励。
- 购买 / 复投按钮。

### 购买弹窗展示内容

```txt
[矿机名称]
[矿机价格]
[最大产出]
[购买方式选择]
[账户余额抵扣金额]
[钱包需支付金额]
[确认购买按钮]
```

购买方式：

```txt
账户余额
钱包余额
账户余额 + 钱包余额
```

### 购买后的状态展示

购买流程中需要展示：

```txt
生成签名中
等待钱包确认
等待链上确认
等待后端同步
购买完成
```

## 3. 市场页

### 布局

```txt
[顶部二级 Tab]
买入市场 | 卖出市场 | 我的订单

[订单列表]
[创建订单入口]
```

### 买入 / 卖出市场

展示公开订单簿。

订单卡片展示内容：

- 买卖方向。
- 订单号缩略。
- 挂单人地址缩略。
- SPACE 数量。
- 剩余 SPACE 数量。
- 价格。
- 创建时间。
- 成交按钮。
- 查看详情入口。

### 创建订单入口

页面提供“创建订单 / 挂单”入口。

创建订单弹窗展示内容：

- 订单方向：买入 / 卖出。
- SPACE 数量输入。
- 价格输入。
- 是否公开。
- 预计 USDT 数量。
- 预计手续费。
- 确认创建按钮。

### 成交订单弹窗展示内容

- 订单号。
- 订单方向。
- 剩余数量。
- 成交数量输入。
- 价格。
- 预计支付或获得金额。
- 确认成交按钮。

### 我的订单区域

二级 Tab：

```txt
未完成
全部
```

未完成订单展示：

- 当前用户 `status = open` 的订单。
- 包含 `visible = false` 的订单。
- 支持买单 / 卖单筛选。

全部订单展示：

- 当前用户所有历史订单。
- 支持买单 / 卖单筛选。
- 支持状态筛选：`open`、`filled`、`cancelled`。

每个订单卡片展示：

- 订单号缩略。
- 买卖方向。
- SPACE 数量。
- 剩余数量。
- 价格。
- 状态。
- 创建时间。
- 取消订单按钮，仅 `open` 状态展示。

## 6. 我的页

### 布局

```txt
[账户信息卡片]
[常用操作列表]
```

### 账户信息展示内容

- 钱包地址。
- 推荐码。
- VIP 等级。
- 节点等级。
- 账户余额。
- 注册时间。

### 操作列表

- 复制钱包地址。
- 复制推荐码。
- 同步节点等级。
- 领取手续费免除资格。
- 查看资金记录。
- 查看我的订单。
- 退出登录。

### 展示规则

- `nodeLevel < 4` 时，手续费免除资格入口隐藏或置灰。
- 同步节点等级后刷新账户信息。
- 领取手续费免除资格后，引导用户调用链上交易。

## 通用状态

### 加载状态

需要展示加载状态的区域：

- 页面首次加载。
- 列表分页加载。
- 表单提交。
- 链上交易处理中。
- 后端同步处理中。

### 空状态

以下列表需要空状态：

- 矿机列表。
- 市场订单列表。
- 我的订单列表。
- 团队直推列表。
- 资金记录列表。

空状态需要展示：

```txt
[空状态说明]
[下一步操作入口]
```

### 错误状态

需要展示错误信息的场景：

- 钱包未连接。
- 签名失败。
- 链上交易失败。
- 后端接口失败。
- 表单校验失败。
- 后端同步超时。

### 交易状态

所有链上交易统一展示以下状态：

```txt
等待钱包确认
交易已提交
等待链上确认
同步后端数据
处理完成
处理失败
```

失败时提供：

```txt
重试
关闭
复制错误信息
```

## 页面数据刷新

### 矿机购买完成后

刷新：

- `GET /miner`
- `GET /miner/my`
- `GET /auth/profile`
- `GET /account/balance-logs`

### 提现完成后

刷新：

- `GET /auth/profile`
- `GET /account/balance-logs`

### 市场订单处理完成后

刷新：

- `GET /market/orders`
- `GET /market/my-open-orders`
- `GET /market/my-orders`
- `GET /market/orders/:id`

### 同步节点等级后

刷新：

- `GET /auth/profile`

## 页面优先级

第一阶段：

- 登录 / 注册页
- 矿机页
- 资产页
- 市场页
- 我的订单区域

第二阶段：

- 团队页
- 我的页

第三阶段：

- 订单详情弹窗
- 团队详情
- 交易状态记录
